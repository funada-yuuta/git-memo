git init

リポジトリの作成

git add

ステージング

git add .

リポジトリ内のすべてのファイルをステージング

git add ./sample.txt

sample.txtファイルのみステージング

git commit

コミット

git commit -m "コメント"

コメントの追加

git commit --amend -m "コメント"

直前のコミットのコメントの変更

git branch

ブランチの作成

git branch

ブランチ一覧と現在のブランチの表示

git branch develop

developブランチの作成

git branch -d feature

featureブランチを削除

git branch -m (master) main

ブランチ名の変更

git switch

作業中のブランチの切替

git switch develop

作業中のブランチをdevelopに切替

git switch -c feature

featureブランチを作成して切替

git reset

コミット・ステージングの取り消し

git reset

ステージング(git add)の取り消し

git reset --hard HEAD^

ひとつ前のコミットへ戻る(現在のステージング、変更、ひとつ前のコミットの内容の取り消し)

git reset HEAD^

ひとつ前のコミットへ戻る(変更点はそのまま)

git checkout

変更内容の取り消し

git merge

マージ

git merge develop

現在のブランチにdevelopをマージ

git diff

ステージングからの変更点を表示

git log

コミットのログを表示

git log --oneline

ログを1行で表示

git remote

リモートリポジトリの設定ん

git remote add origin git@github.com:user-name/repository-name.git

リモートリポジトリの追加

git remote -v

追加したリモートリポジトリの表示

git remote rm origin

リモートリポジトリの削除

git push

リモートリポジトリへプッシュ

git push origin main

mainブランチをリモートリポジトリへプッシュ

git pull

リモートリポジトリからプル

git pull origin main

mainブランチをリモートリポジトリからプル

SSHキーの生成

自PCのコマンドプロンプトでssh-keygenコマンドを使用して、SSH公開鍵と秘密鍵を生成する

ssh-keygen -t ed25519 -C "user@example.com"

公開鍵のコピー

C:\Users\user\.ssh内のid_ed25519.pubの内容をコピー

公開鍵をGithubに設定

  • Githubの右上のアイコンからSettingsをクリック
  • 左メニューのSSH and GPG keysをクリック
  • New SSH keyをクリック
  • タイトル、公開鍵の内容を入力してAdd SSH keyをクリック

接続確認

  • 自PCのコマンドプロンプトでコマンドを入力し接続確認
  • ローカルリポジトリ(.gitがあるフォルダ内)で行う
  • successfullyが出たら成功

ssh -T git@github.com

作業用ブランチの作成

作業用のブランチを作成し、変更、ステージング、コミットはそこで行う

git switch -c develop
git add .
git commit -m "message"

作業用ブランチをリモートリポジトリへプッシュ

git push origin develop

プルリクエスト発行

  • Github上でCompare & pull requestボタンをクリック
  • マージ元、マージ先のブランチを指定(main ← develop)
  • 適宜メッセージを入力
  • Create pull requestをクリック

マージ

  • 適宜コメントを入力
  • Merge pull requestをクリック
  • リモートリポジトリ上の作業用ブランチを削除

ローカルリポジトリにプル

  • リモートリポジトリの本番用ブランチをローカルリポジトリの本番用ブランチにプル
  • ローカルリポジトリの作業用リポジトリを削除

git pull origin main
git branch -d develop
